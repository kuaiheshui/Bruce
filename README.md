# Bruce
game assistant

自走棋AI助手项目 - 技术摘要与规划
📌 项目目标
开发一个基于纯视觉方案的AI，能够自动操作自走棋类游戏。核心流程为：感知游戏画面 -> 决策游戏策略 -> 执行鼠标键盘操作。

🏗️ 核心架构：三层模型
感知层 (Perception)：负责获取并解析游戏画面，输出结构化游戏状态。

决策层 (Decision)：根据游戏状态，制定购买、升级、布阵等策略。

执行层 (Execution)：将策略转化为精确的鼠标键盘操作，作用于游戏。

🔧 技术选型与路径
1. 感知层（当前重点）
方案：纯视觉方案。优先通过软件（如FFmpeg）模拟获取屏幕画面，外部摄像头作为备选。

技术栈：

语言：Python（快速原型验证）。

核心库：OpenCV（图像处理、模板匹配）、Tesseract-OCR（识别数字/文字）、PyAutoGUI（后期模拟操作）。

开发数据：使用静态游戏截图进行算法开发与测试。

实施路径：

预处理与定位：用OpenCV定位固定UI区域（金币区、商店区、棋盘）。

信息提取：OCR识别数字；模板匹配识别图标；后期可引入目标检测（YOLO）识别英雄。

状态结构化：将信息组织为程序可读的对象（如JSON）。

2. 决策层
初期：实现基于规则的引擎（if-then逻辑）。

后期：可探索强化学习（如AlphaZero简化版）等高级算法。

3. 执行层
库选择：PyAutoGUI（简易）或 pynput（更可控）。

关键：将决策转化为抽象的点击、拖动函数，并加入随机延迟等防检测措施。
